Windows

WebUSB should work out of the box in Windows and Chrome. No additional configuration is required.

Debian / Ubuntu

Create a file with udev rules:

sudo touch /etc/udev/rules.d/99-alpakka.rules

Add 2 lines to the file:

{{'SUBSYSTEMS=="usb", ATTRS{idVendor}=="045e", MODE="0660", GROUP="plugdev", SYMLINK+="webusb"'}} {{'SUBSYSTEMS=="usb", ATTRS{idVendor}=="0170", MODE="0660", GROUP="plugdev", SYMLINK+="webusb"'}}

Reconnect the controller and verify the rule has been loaded by checking if /dev/webusb exist.

Steam Deck

Go into desktop mode.

In the terminal app, set a password for your user:

passwd

Type a new password (it won't be visible while typing). Do not lose this password, you may need in the future and it is not easy to recover!.

Create a file with udev rules:

sudo touch /etc/udev/rules.d/99-alpakka.rules

Add 2 lines to the file:

{{'SUBSYSTEMS=="usb", ATTRS{idVendor}=="045e", MODE="0660", GROUP="deck", SYMLINK+="webusb"'}} {{'SUBSYSTEMS=="usb", ATTRS{idVendor}=="0170", MODE="0660", GROUP="deck", SYMLINK+="webusb"'}}

Reconnect the controller and verify the rule has been loaded by checking if /dev/webusb exist.

Known issues: The controller has to be connected before starting Chrome/Chromium, if the controller is restarted then the browser must be restarted too. If you are an Arch expert and figure out go to fix this please let us know.

Other platforms

WebUSB also seems to work out of the box in Android and MacOS, though we are not officially testing these
